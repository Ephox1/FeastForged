## One-line GitHub Description

## Short Portfolio Blurb

FeastForged is a product-style Flutter application I built around real user flows rather than static UI work. It combines Supabase authentication, SQL migrations, recipe and planner data models, community interactions, and mobile debugging into one end-to-end app. I used Riverpod and GoRouter for state and navigation, iterated against a real Android device, and worked through runtime issues, schema alignment, and launch-readiness polish as the product evolved.

## Slightly Longer Portfolio Blurb

FeastForged is a Flutter + Supabase mobile app I built to explore full-stack product engineering in a mobile context. The app includes authentication, onboarding, dashboard progress tracking, recipe discovery, weekly meal planning, recipe logging, community save/rating flows, and supporting database migrations. Beyond the feature set, the project reflects real engineering iteration: I moved away from fragile codegen paths when toolchain compatibility broke down, aligned the app with a live Supabase schema, debugged Flutter runtime/layout issues on a physical Android device, and improved launch readiness with better UX states, feedback, and planner workflows.

## Interview Talking Points

- Why Flutter for this product
  - Fast iteration across UI, navigation, and product flows with a strong developer loop.

- Why Supabase
  - It gave me auth, Postgres, and a migration-friendly backend that still let me reason directly about schema and SQL.

- Biggest engineering challenge
  - Keeping the app and backend aligned while evolving the product, especially around routing, planner/logging flows, and real-device runtime issues.

- Biggest product lesson
  - A repo becomes much more convincing when it shows complete user flows, realistic data, and thoughtful recovery/empty states instead of just screens.
